# 8. Glossar

## A

**API (Application Programming Interface)**
Programmierschnittstelle zur Interaktion mit dem System über HTTP-Requests.

**Arc42**
Standardisierte Vorlage für Architekturdokumentation von Softwaresystemen.

**Authentifizierung**
Prozess der Identitätsprüfung eines Benutzers oder Systems.

**Autorisierung**
Prozess der Zugriffskontrolle auf Ressourcen basierend auf Berechtigungen.

## B

**BASIC (ZUGFeRD-Profil)**
Einfachstes ZUGFeRD-Profil mit minimalen Pflichtangaben.

**B2G (Business-to-Government)**
Geschäftsbeziehungen zwischen Unternehmen und öffentlicher Verwaltung.

## C

**COMFORT (ZUGFeRD-Profil)**
Mittleres ZUGFeRD-Profil mit erweiterten Informationen für die Buchhaltung.

**Code Coverage**
Metrik für den Anteil des durch Tests ausgeführten Codes.

**CRUD**
Create, Read, Update, Delete - Grundlegende Datenbankoperationen.

## D

**Django**
Python Web-Framework für die Backend-Entwicklung.

**Django REST Framework (DRF)**
Erweiterung für Django zur Erstellung von RESTful APIs.

**DSGVO (Datenschutz-Grundverordnung)**
EU-Verordnung zum Schutz personenbezogener Daten.

## E

**e-Rechnung (Elektronische Rechnung)**
Rechnung in einem strukturierten elektronischen Format.

**EN 16931**
Europäische Norm für das semantische Datenmodell elektronischer Rechnungen.

**ERP (Enterprise Resource Planning)**
Unternehmens-Software zur Ressourcenplanung.

**EXTENDED (ZUGFeRD-Profil)**
Umfangreichstes ZUGFeRD-Profil mit vollständigen Geschäftsdaten.

## G

**GoBD (Grundsätze zur ordnungsmäßigen Führung und Aufbewahrung von Büchern)**
Deutsche Richtlinien für elektronische Buchführung und Aufbewahrung.

## H

**HTTPS (Hypertext Transfer Protocol Secure)**
Verschlüsselte HTTP-Verbindung über TLS/SSL.

## J

**JSON (JavaScript Object Notation)**
Leichtgewichtiges Datenformat für strukturierte Daten.

**JWT (JSON Web Token)**
Standard für sichere Übertragung von Informationen als JSON-Objekt.

## O

**ORM (Object-Relational Mapping)**
Technik zur Abbildung von Objekten auf relationale Datenbanken.

**OpenAPI**
Spezifikation zur Beschreibung von RESTful APIs (früher: Swagger).

## P

**PDF/A-3**
Archivierungsformat für PDFs mit Unterstützung für eingebettete Dateien.

**PostgreSQL**
Open-Source relationale Datenbank.

## R

**req42**
Strukturiertes Framework für Anforderungsmanagement in Anlehnung an arc42.

**REST (Representational State Transfer)**
Architekturstil für verteilte Systeme, insbesondere Web-Services.

**RFC 7807**
Standard für Problem Details in HTTP APIs.

## S

**Schema (XML/JSON Schema)**
Definition der Struktur und Validierungsregeln für Daten.

**Soft Delete**
Löschung durch Markierung statt physischer Entfernung aus der Datenbank.

## U

**Use Case**
Beschreibung einer Systemfunktion aus Anwendersicht.

**User Story**
Kurze Beschreibung einer Anforderung aus Nutzerperspektive.

## X

**XRechnung**
Deutscher Standard für elektronische Rechnungen, basierend auf EN 16931.

**XSD (XML Schema Definition)**
Sprache zur Definition der Struktur von XML-Dokumenten.

**XML (Extensible Markup Language)**
Markup-Sprache zur Darstellung strukturierter Daten.

## Z

**ZUGFeRD (Zentraler User Guide des Forums elektronische Rechnung Deutschland)**
Deutscher Standard für elektronische Rechnungen mit PDF und XML.

**ZUGFeRD-Profil**
Spezifische Ausprägung des ZUGFeRD-Standards (BASIC, COMFORT, EXTENDED).
