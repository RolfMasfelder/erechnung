# 2. Randbedingungen

## 2.1 Technische Randbedingungen

| Randbedingung | Erläuterung |
|---------------|-------------|
| Python 3.11+ | Implementierungssprache für Backend |
| Django Framework | Web-Framework für API und Business Logic |
| PostgreSQL | Relationale Datenbank für Datenpersistenz |
| Docker | Containerisierung für Deployment |
| Kubernetes | Container-Orchestrierung für größere Installationen |
| Linux | Ziel-Betriebssystem für Produktion |

## 2.2 Organisatorische Randbedingungen

| Randbedingung | Erläuterung |
|---------------|-------------|
| Entwicklungsprozess | Agile Entwicklung, kontinuierliche Integration |
| Versionskontrolle | Git-basiertes Repository |
| Testing | Mindestens 80% Code Coverage |
| Dokumentation | Arc42 für Architektur, req42 für Anforderungen |

## 2.3 Rechtliche Randbedingungen

| Randbedingung | Erläuterung |
|---------------|-------------|
| § 14 UStG | Pflichtangaben auf Rechnungen, insbesondere fortlaufende Rechnungsnummer (§ 14 Abs. 4 Nr. 4 UStG) zur eindeutigen Identifizierung der Rechnung |
| § 14b UStG | Aufbewahrungspflicht für Rechnungen (10 Jahre) |
| E-Rechnungsverordnung | Verpflichtung zur elektronischen Rechnung im B2G-Bereich |
| EU-Richtlinie 2014/55/EU | Europäischer Standard für elektronische Rechnungsstellung |
| DSGVO | Datenschutz-Grundverordnung für Umgang mit personenbezogenen Daten |
| GoBD | Grundsätze zur ordnungsmäßigen Führung elektronischer Bücher |

## 2.4 Konventionen

### Coding Standards
- PEP 8 für Python-Code
- Type Hints in Python
- Comprehensive Docstrings

### API-Design
- RESTful Principles
- JSON als Datenformat
- Versionierung über URL-Pfad

### Sicherheit
- JWT für Authentifizierung
- HTTPS für alle Verbindungen
- Verschlüsselte Passwort-Speicherung
